# Translations (admin UI)

The admin UI is bilingual: Russian (default) and English. Strings live in
`app/admin/translations/<locale>/LC_MESSAGES/messages.po`. Source strings (`msgid`)
are English; `ru` holds the Russian. Catalogs are compiled in memory at startup —
there are no `.mo` files to commit.

## Add or update a translation

1. Extract strings:  `pybabel extract -F babel.cfg -o messages.pot app`
2. Update an existing locale:  `pybabel update -i messages.pot -d app/admin/translations`
   or init a new one:  `pybabel init -i messages.pot -d app/admin/translations -l <code>`
3. Edit the locale's `messages.po` (fill `msgstr`), then restart the service.
4. For a brand-new language, add its code to `SUPPORTED` in `app/admin/i18n.py`.

The language switcher (RU/EN links in the admin header) sets a `lang` cookie.
